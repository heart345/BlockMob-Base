## 未解 bug / 风险

- **Zombie 程序化动画全是初值待实测**：双足摆轴默认 roll（Angle 第三分量，对齐羊腿；僵尸 leg/arm 局部轴若不同会变左右摆/乱转，需换 pitch/yaw 或符号）；手臂前伸 `BipedArmForwardAngle=-60`（roll，方向/角度猜测）；攻击前挥关键帧（roll 偏移）；死亡侧倒 yaw（复用羊 root 实测，僵尸 root 同 90°X bind 应一致但需确认）。都按游戏表现迭代，和羊腿摆/吃草/侧倒一样多轮调。僵尸家族手臂垂下是转换器全局改动（zombie/husk/drowned/pigman/baby_zombie），skeleton/wither 仍前伸（未做）。
- **空中/跳跃中死亡尸体悬空**：死亡 `SetSolid(SOLID_NONE)` 后靠重力落地会穿地，故死亡后每 tick `SetVelocity(0)+SetGravity(0)` 让尸体静止在死亡位置——空中或跳跃中死的会悬空倾倒。主流地面死亡正确。若要空中死落地，需在保持 solid（或单独 ground trace 落点 + 下沉补偿）下做，避免穿地。
