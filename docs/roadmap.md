# 路线图

## VJ Base 是干什么的

VJ Base 是 Garry's Mod 的 SNPC 框架。它提供很多 NPC 底层能力：

- 出现在生成菜单里
- 阵营和敌友关系
- 索敌和视野判断
- 行为调度和战斗辅助
- 生命值、伤害、死亡、尸体、掉落
- 声音钩子
- 动画 activity 辅助
- 配置菜单和管理选项

它不是“让移动自然”的魔法层。移动僵硬通常来自 Source 寻路限制、模型和动画不匹配、动画播放速率不对、缺少转向/停步过渡，以及 AI 每帧频繁切换目标。

## VJ Base 还是自研 Base

适合用 VJ Base 的情况：

- 想快速兼容 Workshop 常见生态
- 可以接受 VJ 的行为模型
- 需要很多通用 NPC 功能马上可用

适合自研 Base 的情况：

- 想把行为尽量做成 Minecraft 逻辑
- 想直接控制寻路、脚步声、攻击时机、消失规则和生物状态
- 能接受更长的工程周期

推荐路线：先做一个小型 NextBot 原型，再和 VJ 版本对比。如果原型移动和行为明显更像 Minecraft，就继续自研；如果 VJ 的某些基础设施有价值，只抽取不影响手感的部分思路。

## 第一条纵向切片

生物：Zombie。

需要验证的手感：

- 没目标时会停顿和游荡，而不是一直滑行
- 玩家靠近时能发现目标
- 追击时不会频繁抽搐转向
- 只在 Minecraft 风格的短距离内攻击
- 攻击有冷却
- 有待机声、脚步声、受伤声和死亡声
- 日光燃烧后面再做，先把基础战斗手感做好

## 测试地图清单

先用平地图，再用有障碍的地图。

1. 生成一只僵尸，玩家不靠近：应该待机和游荡。
2. 玩家走近：应该发现并追击。
3. 玩家绕圈横移：应该能转向，但不能疯狂旋转。
4. 让它打中玩家：伤害节奏应该像 Minecraft，而不是 HL2 僵尸。
5. 玩家绕到简单障碍后：寻路应该能恢复。
6. 生成五只僵尸：它们不应该完全挤在同一条路径上。
7. 杀死一只：死亡声和清理应稳定。

## 生物扩展顺序

Zombie 做顺以后：

1. 被动陆地生物：Pig 或 Cow。
2. 远程敌对生物：Skeleton。
3. 特殊规则高个生物：Enderman。
4. 飞行生物：Phantom 或 Ghast。
5. 水生生物。
6. Boss 或复杂生物。

每新增一个类型，都只给共享 Base 增加一种真正需要的新能力。

## 资产管线

预期流程：

1. 用 Blockbench 或类似工具整理模型几何。
2. 做 Source 可用的骨骼和动画片段。
3. 导出 SMD/DMX。
4. 通过 Crowbar/StudioMDL 和 QC 文件编译。
5. 把编译出的 MDL/VTX/VVD/PHY 放到 `models/mcgm/`。
6. 把声音放到 `sound/mcgm/`。
7. 用 Lua 实体配置注册。

公开发布 Workshop 时，不要直接分发 Minecraft 原版资产，除非授权路径很明确。
